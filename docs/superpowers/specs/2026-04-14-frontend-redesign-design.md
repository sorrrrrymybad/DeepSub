# DeepSub 前端页面重构设计文档

**日期：** 2026-04-14
**作者：** Sorrymybad
**状态：** 已批准，待实现

---

## 1. 目标

在**不改变现有功能与路由**的前提下，参考 `/Users/dev/Downloads/Cli-Proxy-API-Management-Center-main` 的页面风格与元素逻辑，重构 DeepSub 当前前端页面，使其整体呈现更完整的后台管理台体验。

本次重构的重点是：

- 重建全站视觉语言与布局骨架
- 将当前页面从“功能页直出”改造为“管理台信息层级”
- 引入三套可切换主题
- 将语言切换与主题切换改为参考项目风格的图标菜单
- 为设置页补充页面内部目录导航

---

## 2. 范围与边界

### 2.1 在范围内

- 重构 `frontend` 内现有页面的视觉样式与布局结构
- 重构全局 `Layout`
- 新增主题切换能力
- 抽取新的页面级通用组件
- 优化空态、加载态、错误态的统一呈现
- 优化桌面端与移动端布局退化逻辑

### 2.2 不在范围内

- 不新增业务路由
- 不新增业务接口
- 不调整现有任务、设置、历史的核心功能逻辑
- 不修改后端 API 语义
- 不将项目整体改成参考项目的页面结构映射

---

## 3. 已确认的设计决策

### 3.1 总体方向

- 保留 DeepSub 当前功能和路由
- 仅重构视觉风格、布局层级、信息密度和交互容器
- `Tasks` 页面承担“准首页”角色

### 3.2 主题策略

系统提供三套手动主题：

- `A`：暖灰玻璃管理台
- `B`：深色运维控制台
- `C`：白底企业面板

同时支持一个主题模式：

- `auto`：跟随系统明暗偏好作为默认行为

行为规则：

- 首次进入默认 `auto`
- 用户手动切换到 `A/B/C` 后，本地持久化
- 刷新页面后保留用户选择
- 主题切换通过 CSS Variables 驱动，而不是通过大量硬编码 class 分支

### 3.3 顶栏交互

以下交互改为参考项目风格：

- 语言切换：图标按钮 + 弹出菜单
- 主题切换：图标按钮 + 弹出菜单

不再使用当前直接显示文字按钮的形式。

主题菜单项包含：

- 跟随系统
- A 暖灰玻璃
- B 深色运维
- C 白底企业

语言菜单项包含：

- 中文
- English

## 4. 顶层布局设计

全站采用统一的管理台骨架：

- 左侧导航栏
- 右侧主内容区
- 内容区顶部操作栏
- 页面内部 hero / 统计卡 / 内容面板

### 4.1 左侧导航栏

保留现有路由入口：

- 任务列表
- 新建任务
- 系统设置
- 历史记录

导航样式参考项目的管理台侧栏逻辑：

- 品牌区位于顶部
- 当前页高亮
- 整体采用卡片化、低对比暖灰/深色/白底主题风格
- 移动端支持折叠或顶部触发打开

### 4.2 顶部操作栏

顶部操作栏承载：

- 刷新或全局操作入口
- 语言图标菜单
- 主题图标菜单

风格要求：

- 图标化
- 浮层菜单
- 与参考项目同类视觉层级

### 4.3 页面壳统一

页面内部将统一采用以下结构组合：

- `PageHero`
- `StatCard`
- `SectionCard`
- `FilterTabs`
- `EmptyState`

这样避免每个页面独立散写容器和样式。

## 5. 页面重排方案

## 5.1 TasksPage

`TasksPage` 改造为“准首页”。

### 页面层级

从上到下为：

1. Hero 区
2. 统计卡区
3. 状态筛选与批量操作工具栏
4. 任务列表
5. 分页区

### Hero 区内容

- 当前任务总览
- 欢迎信息或状态标题
- 当前系统运行状态摘要
- 主操作入口，例如“新建任务”

### 统计卡内容

建议展示：

- 全部任务数
- 运行中任务数
- 已完成任务数
- 失败任务数

### 保留行为

以下功能保持不变：

- 状态筛选
- 批量取消运行任务
- 自动轮询列表
- 任务日志抽屉
- 失败重试
- 分页

### 目标

让 `TasksPage` 从“单纯列表页”升级为“任务管理首页”。

---

## 5.2 NewTaskPage

`NewTaskPage` 从长表单改造成步骤感更强的任务配置页。

### 桌面端布局

采用两栏布局：

- 左栏：服务选择与文件浏览
- 右栏：参数配置、已选文件摘要、提交区域

### 信息顺序

1. 选择 SMB 服务
2. 浏览并选择文件
3. 配置源语言、目标语言、STT 引擎、翻译引擎、覆盖策略
4. 查看已选文件摘要
5. 提交任务

### 目标

- 保留原有逻辑
- 强化用户的操作顺序感
- 减少“滚动找控件”的体验

### 移动端退化

移动端自动退化为单列结构，不强行保留两栏。

---

## 5.3 SettingsPage

`SettingsPage` 改造成分区明确的设置工作区，并新增页面内部设置目录。

### 页面结构

从上到下为：

1. 设置概览区
2. 页面内部目录导航
3. SMB 管理区
4. STT 配置区
5. 翻译配置区

### 设置目录

目录项包括：

- 概览
- SMB 服务器
- STT 配置
- 翻译配置

### 桌面端交互

- 左侧为粘性目录
- 右侧为内容区
- 点击目录项后滚动到对应 section
- 当前 section 具备高亮态

### 移动端交互

目录退化为：

- 顶部 tabs
或
- 顶部下拉目录

具体实现选择以更稳定为准，但语义仍是“页面内部目录”。

### SMB 区域

分成两个子块：

- 已配置 SMB 服务器列表
- 新增 SMB 服务器表单

### STT 与翻译配置区域

不再直接表现为连续长表单，而是作为独立的配置卡片区呈现。

### 目标

使设置页更接近参考项目的“设置中心”体验，而不是线性堆叠表单。

---

## 5.4 HistoryPage

`HistoryPage` 保持轻量，不做过重首页化处理。

### 页面层级

1. 归档说明头部
2. 历史任务列表
3. 分页区

### 补充信息

新增轻量摘要：

- 已完成任务总数
- 当前分页位置

### 目标

让其成为“历史归档视图”，避免与 `TasksPage` 的实时管理角色重复。

## 7. 视觉系统要求

### 7.1 主题令牌

所有主题都应覆盖以下 token：

- 页面背景
- 卡片背景
- 次级背景
- 边框
- 主文字
- 次文字
- 强调色
- 成功色
- 警告/错误色
- 阴影
- 浮层背景
- 浮层边框

### 7.2 视觉风格约束

- 不使用当前默认的紫色主色
- 不继续沿用当前过于简陋的基础灰白风格
- A 主题尽量靠近参考项目的暖灰、轻玻璃、柔和边界
- B 主题强调运维与监控感
- C 主题强调企业后台的明亮与规整

### 7.3 字体与层级

当前 `Inter` 可保留，但页面层级必须重做：

- Hero 标题更强
- 统计数字更突出
- 标签/eyebrow 更明确
- 表单标签与正文层级拉开
